Manage Users

Control who can access the CMS via Google Sign-In.

Google Sign-In Setup To enable Google login, set the GOOGLE_CLIENT_ID environment variable in Hostinger's Node.js settings. Get your Client ID from Google Cloud Console → APIs & Services → Credentials → Create OAuth 2.0 Client ID (Web application). Add https://akadigital.net as an Authorised JavaScript Origin. Anyone signing in with an unknown Google account creates an access request — approve or reject it here or on the Dashboard. On Hostinger also set CMS_USERS_FILE to a path outside the app directory (e.g. /home/<user>/cms_users.json) so the approved-user list survives Git deploys.

Permanent super admins Set SUPER_ADMIN_EMAILS (comma-separated Gmail addresses) to guarantee those accounts always have super access, even if cms_users.json is ever lost or edited — shown below with a 🔒 Reserved badge. Set DISABLE_PASSWORD_LOGIN=true once every super admin can sign in with Google to retire the shared admin/team password; flip it back at any time (no redeploy needed) as an emergency fallback.
